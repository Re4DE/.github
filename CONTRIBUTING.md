# Contributing to Re4DE

Thank you for your interest in Re4DE.

Re4DE is a research-driven software project that explores dataspace-based integration concepts for the German energy system. While the repositories are publicly available, Re4DE is not operated as a continuously managed open-source community.

Nevertheless, feedback, questions, and selected contributions are welcome.

---

## Community and Support Expectations

- This is not an actively managed community repository.
- Issues and pull requests are reviewed selectively.
- There is no guarantee of response times, long-term maintenance, or backward compatibility.

---

## How to Engage

### Questions about the Code or Change Proposals

If you have questions about the codebase or proposals for changes, please contact the maintainers listed in the organization README or open an issue to discuss the topic before starting larger work.

### Questions about the Project or Dataspaces in the German Energy System

For questions related to the Re4DE project context, dataspaces, or the German energy system in general, please contact the Future Energy Lab.

---

## What Contributions Are Welcome

The following types of contributions are welcome:

- Bug reports and improvement suggestions
- Clarifications or extensions to documentation
- Conceptual feedback on architecture or design decisions
- Code contributions after prior coordination

Please discuss larger changes in advance to align on scope, approach, and repository fit.

---

## Development Workflow (for Code Contributions)

1. Fork the repository.
2. Create a feature branch based on `main`.
3. Implement your changes in focused, comprehensible commits.
4. Ensure the project builds successfully and that relevant tests pass.
5. Open a Pull Request with a clear description of motivation and scope.

Pull Requests may be reviewed selectively and merged at the discretion of the maintainers.

---

## Code Style and Quality

Please follow the existing conventions in the repository and keep contributions minimal and well-scoped.

If tests, documentation, or interfaces are affected, update them accordingly.

---

## Licensing

Ensure that you have the right to contribute the code or content, especially if you are acting on behalf of an organization.
